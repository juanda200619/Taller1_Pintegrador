Objetivos del Proyecto

Metas que guían el desarrollo de UniTicket

Objetivo General

Diseñar y desarrollar un sistema web llamado UniTicket que permita la gestión eficiente de incidencias técnicas en los laboratorios universitarios, integrando prácticas DevOps, automatización de procesos y despliegue continuo.

Objetivos Específicos

  • build Implementar prácticas DevOps
    Aplicar las diferentes prácticas de DevOps (Integración, Entrega y Despliegue Continuo), reconociendo sus beneficios y retos para agilizar el ciclo de desarrollo del software UniTicket.
  • storage Gestionar base de datos
    Configurar una base de datos relacional para almacenar los tickets y su información.
  • cloud Desplegar en la nube
    Publicar la aplicación en un servidor VPS usando contenedores Docker.
  • verified Garantizar calidad del software
    Aplicar pruebas automatizadas y análisis de código para asegurar calidad y estabilidad.
  • group_work Fomentar trabajo colaborativo
    Utilizar metodologías de control de versiones como Gitflow para el desarrollo en equipo.
